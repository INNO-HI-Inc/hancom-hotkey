특징 #

▲ 네이티브 Swift

Hammerspoon 없이도 메뉴바에 가볍게 상주. CPU/메모리 부담 최소.

▲ 한컴 3종 지원

한글, 한셀, 한쇼 모두 자동 감지.

▲ 전역 토글

⌃⌥⌘P로 어디서든 즉시 on/off.

▲ 안전 설계

암호 입력창(Secure Input)에서는 자동 비활성. 앱이 비활성일 땐 키 변환 안 함.

▲ 자동 업데이트

Sparkle 통합. 버그 수정과 기능 개선이 즉시 전달.

▲ 공증 완료

Apple Developer ID 서명 + 공증. Gatekeeper 경고 없이 바로 설치.

기본 매핑 #

누르는 키HWP에 전송스타일 목록 위치예시 스타일명
⌘ 0⌃ 11번째바탕글
⌘ 1⌃ 22번째본문
⌘ 2⌃ 33번째제목 1
⌘ 3⌃ 44번째제목 2
⌘ 4⌃ 55번째제목 3

※ "예시 스타일명"은 한컴 오피스 기본 템플릿 기준입니다. 한컴은 스타일 이름이 아니라 목록 순서로 단축키를 붙이므로, 원하는 스타일을 ⌘0~4에 배치하려면 HWP 서식 → 스타일에서 목록 순서를 조정하세요. 앱 설정의 단축키 탭에 5단계 가이드가 있습니다.

⌘0~4가 안 먹는다면 — HWP 스타일 5단계 설정 #

한컴 Mac판은 스타일 목록 순서에 따라 자동으로 단축키를 부여합니다. 1번째 = ⌃ 1(보통 바탕글), 2번째 = ⌃ 2, … 본 앱은 ⌘ 0~4를 ⌃ 1~5로 바꿔 보내므로, 1~5번째 자리에 원하는 스타일이 있어야 동작합니다.

  1. 스타일 창 열기 — HWP 상단 메뉴: 서식 → 스타일… (또는 F6, Touch Bar 맥북은 fn + F6)
  2. 옮길 스타일 선택 — 목록에서 자주 쓰는 스타일을 한 번 클릭
  3. 위 / 아래 버튼으로 이동 — 목록 우측의 ▲▼ 버튼 또는 드래그
  4. 1~5번째 자리에 배치 — ⌘ 0은 1번째(바탕글), ⌘ 1은 2번째, …, ⌘ 4는 5번째
  5. 완료 — 창 닫으면 자동 저장. HWP에서 ⌘ 0~4 눌러 확인

예: "본문 가."를 ⌘ 1로 쓰고 싶다면, 스타일 목록의 2번째 자리로 옮기세요. 앱 설정 → 단축키 탭에도 동일한 가이드가 있습니다.

설치 #

  1. 위의 DMG 다운로드 버튼 클릭
  2. DMG 파일 더블클릭 → 한컴단축키.app을 Applications로 드래그
  3. Launchpad에서 한컴단축키 실행
  4. 첫 실행 시 접근성 권한 부여 (아래 설명 참조)
  5. 한글, 한셀, 한쇼에서 ⌘ 0~4 눌러 확인

접근성 권한이 왜 필요한가요? #

▲ 키 입력을 변환하려면 macOS의 접근성 권한이 필수입니다

앱이 ⌘ 0을 감지해서 ⌃ 1로 바꿔 한컴에 보내는 동작 자체가 macOS 기준 "키 후킹"이라, 접근성 권한 없이는 동작하지 않습니다.

  • 한글(HWP)·한셀·한쇼가 활성 상태일 때만 변환합니다
  • 암호 입력창(Secure Input)에서는 자동으로 비활성화됩니다
  • 키로그를 저장·전송하지 않습니다 — 변환만 하고 메모리에서 폐기
  • 오픈소스 + Apple Developer ID 서명 + Apple 공증 완료

설정 경로: 시스템 설정 → 개인정보 보호 및 보안 → 접근성 → 한컴단축키 토글 ON. 앱이 처음 실행되면 안내 화면이 직접 시스템 설정을 열어드립니다.

자주 묻는 질문 #

한 번 누르면 정렬만 바뀌고, 두 번 눌러야 글씨체까지 바뀌어요

한컴 HWP의 의도된 동작이며 본 앱과 무관합니다. 한컴 공식 문서 그대로:

  • 첫 번째 적용: 새로운 스타일 내용대로 바꾸되, 사용자가 별도로 바꾼 글자 모양·문단 모양은 보호해 그대로 둡니다.
  • 두 번째 적용: "본문을 스타일 모양대로 덮어쓸까요?" 메시지가 뜨거나 자동으로 덮어쓰기 → 사용자가 직접 바꾼 글자 모양까지 모두 스타일 정의로 초기화.

즉, 그 문단의 글씨체를 미리 수동으로 바꿔놨던 경우 두 번 눌러야 글씨체까지 바뀝니다. 회피하려면 적용 전에 ⌃ 1(바탕글)로 한 번 초기화하거나, 새 문단에 처음부터 적용하세요. 출처: 한컴 공식 — 스타일 적용하기.

⌘0~4를 눌러도 아무 일도 일어나지 않아요

가장 흔한 원인은 HWP 스타일 목록의 1~5번째 자리가 비어있거나, 시각적 변화가 없는 자동 생성 스타일이라서 그렇습니다.

한컴 Mac은 스타일 목록 순서대로 ⌃ 1 ~ ⌃ 0을 자동 할당합니다. 본 앱은 ⌘ 0~4를 ⌃ 1~5로 변환만 해주므로, 그 자리에 의미있는 스타일이 없으면 결과는 동일하게 무반응입니다.

해결: 위의 5단계 가이드에 따라 자주 쓰는 스타일을 1~5번째 자리로 옮겨주세요.

그 외에도 다음을 확인해 보세요:

  • 메뉴바 아이콘 상태가 ● 활성인가? (⏸ 일시정지 또는 ⚠︎ 권한 필요면 비활성)
  • 접근성 권한이 켜져 있는가? (시스템 설정 → 개인정보 보호 및 보안 → 접근성)
  • 현재 활성 창이 정말 한글·한셀·한쇼인가?
다른 키로 매핑을 바꿀 수 있나요?

현재 버전은 ⌘ 0~4 → ⌃ 1~5 고정 매핑을 제공합니다. 사용자 정의 매핑은 향후 버전에서 검토 중이며, GitHub Issues에 요청 주시면 우선순위가 올라갑니다.

한컴 외 다른 앱의 단축키에 영향을 주나요?

아니요. 한글(HWP)·한셀·한쇼가 활성 창일 때만 키 변환이 동작합니다. Safari, Slack, Xcode 등 다른 앱에서는 ⌘ 0~4가 원래 기능 그대로 작동합니다.

또한 ⌃⌥⌘P로 전체 변환을 즉시 끄거나 켤 수 있습니다.

로그인 시 자동 실행할 수 있나요?

네. 메뉴바 아이콘 → 설정 → 일반에서 "로그인 시 자동 실행" 옵션을 켜면 됩니다. macOS의 SMAppService API를 사용하므로 별도 권한 부여가 필요 없습니다.

자동 업데이트는 어떻게 동작하나요? 끌 수 있나요?

Sparkle 프레임워크 기반으로 24시간마다 새 릴리즈를 자동 확인합니다. 업데이트 발견 시 알림창이 뜨며, 사용자가 수락해야 설치됩니다.

자동 확인을 끄려면 메뉴바 아이콘 → 설정 → 일반 → 자동 업데이트 확인을 토글하세요. 수동 확인은 메뉴바 아이콘 → 업데이트 확인으로 가능합니다.

암호 입력 시 키가 새지 않을지 걱정됩니다.

안심하셔도 됩니다. 암호 입력창(은행 사이트, 1Password, macOS 잠금화면 등)이 활성화되면 macOS가 Secure Input 모드를 켜며, 한컴단축키는 이 모드를 감지해 키 변환을 자동으로 비활성화합니다.

그리고 본 앱은 키 입력 자체를 저장·전송하지 않습니다. ⌘0~4 매칭 후 즉시 ⌃1~5로 변환·전송하고 메모리에서 폐기합니다. 코드는 GitHub에서 공개되어 있어 검증 가능합니다.

삭제하려면 어떻게 하나요?

1) 메뉴바 아이콘 → 종료
2) Applications 폴더에서 한컴단축키.app을 휴지통으로 이동
3) 시스템 설정 → 개인정보 보호 및 보안 → 접근성에서 항목 제거
4) (선택) 설정 데이터 제거: 터미널에서 defaults delete com.innohi.hancomhotkey

Intel 맥에서도 동작하나요?

네. Universal binary로 빌드되어 Apple Silicon(M1/M2/M3/M4) 및 Intel 맥 모두 네이티브로 동작합니다. macOS 13 Ventura 이상이 필요합니다.

한컴 오피스 어떤 버전에서 동작하나요?

한컴오피스 Mac 버전 v12.30 이상에서 동작 확인되었습니다. 이전 버전이나 다른 버전에서도 일반적으로 동작하지만, 한컴이 향후 업데이트로 단축키 체계를 바꾸면 영향이 있을 수 있습니다. 그럴 경우 이슈를 남겨주세요.

호환 #

macOS 13 Ventura 이상. Apple Silicon 및 Intel 맥 모두 지원 (Universal binary).

한컴 오피스 버전: v12.30 이상에서 검증. 다른 버전에서도 일반적으로 동작합니다.
