개인정보처리방침

최종 개정일: 2026-04-29 · 적용 버전: 한컴단축키 v0.5.0

한 줄 요약

한컴단축키는 사용자의 키 입력·문서 내용·개인정보를 수집·저장·전송하지 않습니다. 자동 업데이트 확인을 위해 GitHub 서버에 익명 HTTPS 요청만 발생합니다.

1. 처리하는 정보의 범위

수집하지 않는 정보

  • 이름, 이메일, 전화번호 등 식별정보
  • 키로그(키 입력 기록)
  • 한컴 오피스 내 문서 내용·파일명
  • 접속 IP, 위치 정보, 기기 식별자
  • 사용 통계, 텔레메트리

로컬에서만 처리되는 정보

  • 키 입력 이벤트: macOS 접근성 API로 ⌘0~4를 감지해 즉시 ⌃1~5로 변환·전송하고 메모리에서 폐기. 디스크에 저장하거나 외부로 전송하지 않습니다.
  • 활성 앱 정보: 한글·한셀·한쇼가 활성 창인지 확인하기 위해 macOS의 NSWorkspace API로 활성 앱 번들 ID를 조회합니다. 이 정보 역시 저장·전송되지 않습니다.
  • 앱 설정값: 자동 실행 여부, 자동 업데이트 확인 여부 등 사용자 환경설정은 ~/Library/Preferences/com.innohi.hancomhotkey.plist에 로컬 저장됩니다. 외부 서버로 동기화하지 않습니다.

2. 외부 서버와의 통신

한컴단축키는 다음 두 가지 경우에 한해 외부 서버에 HTTPS 요청을 합니다.

  • 자동 업데이트 확인 (Sparkle): 24시간마다 또는 사용자가 수동 확인 시, GitHub Pages(inno-hi-inc.github.io)에 호스팅된 appcast.xml을 다운로드합니다. 요청에는 macOS 버전과 앱 버전 정도의 표준 User-Agent만 포함되며, 식별 가능한 사용자 정보는 포함되지 않습니다.
  • 업데이트 다운로드 시: 사용자가 업데이트 알림을 수락하면 GitHub Releases(github.com)에서 새 DMG를 다운로드합니다. 이는 GitHub 서비스 이용약관 및 개인정보처리방침의 적용을 받습니다.

자동 업데이트 확인을 끄려면 메뉴바 아이콘 → 설정 → 일반 → 자동 업데이트 확인을 토글하세요.

3. 권한 사용 내역

  • 접근성(Accessibility): 키 입력 후킹 및 변환을 위한 macOS 권한. 이 권한은 키 변환 외 어떤 용도로도 사용되지 않습니다. macOS의 Secure Input 모드(암호 입력창 등) 활성화 시 자동으로 비활성화됩니다.
  • 네트워크: 자동 업데이트 확인용 HTTPS 통신. 그 외 네트워크 활동 없음.

4. 데이터 보관 및 파기

본 앱은 사용자의 개인정보를 수집·보관하지 않으므로 별도 파기 절차가 없습니다. 앱을 삭제하면 로컬 설정값(plist)은 다음 명령으로 함께 제거할 수 있습니다.

defaults delete com.innohi.hancomhotkey

5. 오픈소스 검증

본 앱의 모든 소스코드는 GitHub 공개 저장소에서 확인할 수 있습니다. 위 처리방침의 내용은 코드 수준에서 직접 검증 가능합니다.

6. 변경 통지

본 처리방침이 변경되는 경우, 본 페이지의 "최종 개정일"이 갱신되고 GitHub Releases 변경 이력에 명시됩니다. 중대한 변경 시에는 앱 내 알림으로도 안내합니다.
